树状数组

其实假如你会线段树，并且比较熟练，你可以直接离开。

为什么呢？ 虽然线段树和树状数组的基础功能是差不多的，但是 — 树状数组能做到的操作线段树一定能做到，但是线段树能做的事情树状数组不一定能做到。

虽然话是这么说，但是在实际赛场上，树状数组性价比是很高的。尽管树状数组功能没有线段树那么多，但是树状数组的代码长度以及思维难度都是比线段树低的。

假如题目只需要单点修改的话，树状数组绝对是我十分推荐的选择。

概念

上图为树状数组原理示意图。

设用 aaa 表示原序列，ccc 如图所示。

则我们可以得出：

c1=a1c_1 = a_1c1​=a1​；

c2=c1+a2c_2 = c_1 +a_2c2​=c1​+a2​；

c3=a3c_3 = a_3c3​=a3​；

c4=c2+c3+a4c_4 = c_2 + c_3 + a_4c4​=c2​+c3​+a4​；

c5=a5c_5 = a_5c5​=a5​；

c6=c5+a6c_6 = c_5 + a_6c6​=c5​+a6​；

c7=a7c_7 = a_7c7​=a7​；

c8=c4+c6+c7+a8c_8 = c_4 + c_6 + c_7 + a_8c8​=c4​+c6​+c7​+a8​。

再设 kkk 表示 iii 的二进制中从最低位往高位连续零的长度。

便可以得到 ci=ai−2k+1+ai−2k+2+⋯+aic_i = a_{i - 2^k + 1} + a_{i - 2^k + 2} + \cdots + a_ici​=ai−2k+1​+ai−2k+2​+⋯+ai​。

现在问题来了，如何求出 kkk 呢？最低位往高位连续零的长度其实也相当于求 1 的最低位。所以便有神仙想出了 lowbit(x) = x & (-x)。

为什么可以这样写呢？

分析 x & (-x) 这个式子。

根据计算机补码的性质（补码就是原码的反码加一）：

例如 101010，

原码：(10)10=(1010)2(10)_{10} = (1010)_2(10)10​=(1010)2​；

反码：(0101)2(0101)_2(0101)2​；

补码：(0110)2(0110)_2(0110)2​。

不难发现，反码与原码的每一位都不相同。但是当反码加上一之后，会一直进位直到遇到 000，这个 000 就变成了 111（下文称该 111 为特殊数），这时这个数末尾就构造了一个 10000... 串。由于补码是由反码加一的来的，所以除了变成 111 的 000 以外，其他的数位都是不相等的。所以可以发现特殊数以前的部分补码与原码相反；特殊数的位置补码与原码都是 111；特殊数以后的部分补码与原码都是 000。所以 lowbit(x)\texttt{lowbit}(x)lowbit(x) 就等于 kkk。

这样 kkk 就轻松地求出来了。

基本操作

单点修改，区间查询。

修改

设想一下，要更改 axa_xax​ 的值，就必须要把所有包含了 axa_xax​ 的 ccc 修改。

那我们如何来找到所有包含了 axa_xax​ 的 ccc 呢？我们可以发现一个规律：就是 包含 axa_xax​ 的唯一元素的下标 =ax+lowbit(x)= a_x + \texttt{lowbit}(x)=ax​+lowbit(x)。

查询

利用前缀和的思想，想查询 [l,r][l, r][l,r] 的和，只需要用 sumr−suml−1sum_r - sum_{l-1}sumr​−suml−1​ 就可以了。

所以树状数组的查询实际上就利用了前缀和的思想。与修改相反，查询要从顶端向低端跳，所以相对应的求 [1,x][1, x][1,x] 的区间只要让 xxx 每次都减去 lowbit(x)\texttt{lowbit}(x)lowbit(x) 就行了。

时间复杂度：

  • 构造数组：
    • 暴力：从 111 到 nnn 循环，时间复杂度 O(n)O(n)O(n)。
    • 树状数组：nnn 次修改，时间复杂度 O(nlog⁡n)O(n \log n)O(nlogn)。
  • 修改：
    • 暴力：直接修改，时间复杂度 O(1)O(1)O(1)。
    • 树状数组：从下往上跳，时间复杂度 O(log⁡n)O(\log n)O(logn)。
  • 查询：
    • 暴力：从 lll 到 rrr，时间复杂度 O(n)O(n)O(n)。
    • 树状数组：从上往下跳，时间复杂度 O(log⁡n)O(\log n)O(logn)。
  • 总复杂度：
    • 暴力：O(nm)O(nm)O(nm)。
    • 树状数组：O((n+m)log⁡n)O((n+m)\log n)O((n+m)logn)。

差分树状数组

差分树状数组可以实现区间修改，单点查询。

具体怎么实现呢？其实树状数组的基本操作是不用改的，只是在做操作时调用函数的方式有所不同。并且这些操作用线段树代码非常冗长，但是树状数组却简洁明了。
